Please add a full Persona Booster (“Improve Persona Accuracy”) system to EverSpeak.  
This should audit the selected persona using AI and generate structured improvement recommendations.

Do NOT modify or break any existing features (wizard, journaling, calibration, strict mode, bulk import, voice-to-text, etc.). Only extend.

------------------------------------
A. BACKEND: BOOST ENDPOINT
------------------------------------

Add new endpoint:

POST /api/personas/:id/boost

Behavior:

1) Validate persona exists.
2) Load persona:
   - name
   - relationship
   - description
   - memories[]
   - settings (tone defaults, sliders, boundaries)
3) Load journals where "persona_id" matches requested id.
4) Load chat logs if they exist:
   - If you have a lightweight chat log storage, use it.
   - If not, skip this step and include an empty array.

5) Package all persona data into a single analysis object:

{
  "persona": { ...full persona... },
  "memories": [...],
  "journals": [...],
  "settings": { ... },
  "calibration": { ... },
  "strict_mode": boolean or null,
  "chat_logs": [...]
}

6) Send this object to OpenAI GPT-4o-mini with a carefully constructed prompt:

Prompt must ask the model to analyze the persona and return ONLY JSON with:

{
  "missing_categories": [ "childhood", "humor", ... ],
  "new_memories": [
    {
      "category": "...",
      "text": "...",
      "weight": number
    }
  ],
  "tone_suggestions": {
    "humor": 0-5,
    "honesty": 0-5,
    "sentimentality": 0-5,
    "energy": 0-5,
    "advice_giving": 0-5
  },
  "boundary_flags": [
    "risk of regret spirals",
    "tone too sentimental",
    ...
  ]
}

Rules:
- The model must never claim supernatural info.
- It must only infer based on provided persona data.
- Use firm instruction: “CRITICAL: Do not invent unknown biographical facts. Suggest only memories or traits that could plausibly emerge from patterns in the supplied data.”

7) After receiving the JSON:
   - Validate it (ensure arrays and numbers are present).
   - DO NOT automatically modify the persona.
   - Return:

{
  "success": true,
  "recommendations": { ...parsed model output... }
}
